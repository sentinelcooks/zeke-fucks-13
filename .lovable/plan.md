

## Plan: Rework Analysis Breakdown to Show Data-Driven Comparisons

### Problem
The Analysis Breakdown section renders every factor as "X is essentially even between both teams" or "X slightly favors Y" ‚Äî bland text generated by the `factorToInsight()` function. The user wants it to show actual data comparisons like the props section does.

### Solution
Replace the plain text list with visual factor comparison cards that show each team's score, a comparison bar, and the factor weight ‚Äî similar to how the props section displays breakdowns.

---

### Changes

**1. Update `moneyline-api/index.ts` ‚Äî return `factorBreakdown` directly instead of converting to bland text**
- Remove `factorToInsight()` calls on lines 915-917, 973-975, and 1028-1034
- Instead, pass `factorBreakdown` array as-is (it already has `label`, `team1Score`, `team2Score`, `weight`, `advantageScore`)
- Keep only special factors (injury warnings, writeup, B2B, etc.) in the `factors` array
- For the generic NBA path (line 1028-1034): replace `cleanFactors` with the raw descriptive `analysis.factors` array and include `factorBreakdown` separately

**2. Update `MoneyLineSection.tsx` ‚Äî replace the text list with visual factor bars**
- Redesign the "Analysis Breakdown" section to render `results.factorBreakdown` as comparison bar cards
- Each factor shows: label, weight badge, team1 score bar vs team2 score bar, and which team it favors
- Keep the AI Summary card and any special emoji-prefixed lines as-is
- Color the bars: green for advantage (>55), red for disadvantage (<45), neutral for even
- Sort factors by weight (highest impact first)
